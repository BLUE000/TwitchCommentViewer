# Twitchコメント解析ツール 結合試験仕様書 (Integration Test Specification)

本ドキュメントは「基本設計書（`basic_design.md`）」に対応する結合試験（Integration Test）の仕様を定義する。
UI層、コントローラ層、モジュール層の各コンポーネントが、想定通りにイベント（メッセージパッシング）を用いて連携し、一連の業務フローが完遂できることを検証する。

## 1. 試験アプローチ
* **自動テスト**: QTestを用いたシミュレーション（ボタンクリックのプログラム的発火など）およびGTestを用いた複数モジュール結合状態のテスト。
* **ダミー化 (Mocking)**: 外部通信（Twitchサーバー、OBS、棒読みちゃん）は純粋仮想クラスを用いたMockオブジェクトに差し替え、通信エラーや制限に依存しない環境で結合試験を行う。

## 3. テスト環境・前提条件
* 結合テスト用実行ファイルは、リリースビルドではなくテスト用の構成（Mock注入可能な状態）でビルドされること。
* SQLite用のテストデータベースファイル（インメモリDB等の揮発性環境）を使用し、テストごとに初期化すること。
