# Twitchコメント解析ツール 基本設計書

## 1. システムアーキテクチャ方針

本システムでは、Qtの特性を活かし、UI層と内部ロジック、各連携機能を完全に分離した「メッセージパッシング方式」を採用する。

### 通信プロトコル（QMetaObject::invokeMethodの活用）
* モジュール間やスレッド間の通信において、シグナル＆スロットの静的な `connect` は極力使用しない。
* 代わりに、`QMetaObject::invokeMethod(target, "methodName", Qt::QueuedConnection, ...)` を利用する。
* これにより、関数の呼び出し要求が自動的に「イベント」に変換され、対象スレッドのイベントキューに `postEvent` されるため、非常に疎結合かつスレッドセーフなアーキテクチャが実現する。

## 2. モジュール構成とデータフロー

### 2.1 UI層 (Presentation Layer)
* **構成**: Qtの `.ui` ファイルと対応するUIクラス。
* **役割**:
  * ユーザー操作を起点とし、`on_pushButton_clicked()` 等のイベントハンドラから、コントローラ層へ `invokeMethod` で処理を依頼する（行き）。
  * 内部処理モジュールから `invokeMethod` によって投げられた「UI更新イベント」を受け取り、画面を描画する（帰り）。

### 2.2 コントローラ層 (Core Application Layer)
* **役割**:
  * 各モジュールのインスタンス生成と管理。
  * 各モジュールをバックグラウンドスレッド（`QThread`）に配置し、メインスレッド（UI）のフリーズを防ぐ。

### 2.3 モジュール層 (External & Internal Services Layer)
各機能は完全に独立したモジュールとして実装する。

#### 外部インターフェースモジュール
1. **Twitch受信モジュール (`TwitchEventCollector`)**
   - EventSub(WebSocket)からコメントやイベントを受信し、システムに通知する。
2. **Twitch送信・操作モジュール (`TwitchActionSender`)**
   - コントローラからの要求を受け取り、Twitch API(HTTP)を実行してコメント送信やポイント操作を行う。
3. **OBS連携モジュール (`ObsIntegration`)**
   - OBS操作要求を受け取り、obs-websocket経由でアクションを送信する。
4. **読み上げ連携モジュール (`BouyomiIntegration`)**
   - 棒読みちゃん(TCP)やVOICEVOXへのテキスト送信を担当。

#### 内部処理モジュール
5. **データ管理モジュール (`DatabaseManager`)**
   - `TransCipher-Dist` ライブラリを用いたデータの暗号化・復号と、SQLiteへの書き込み・読み込み。
6. **設定管理モジュール (`ConfigManager`)**
   - Windows DPAPIや `QtKeychain` を利用したセキュアなOAuthトークンの管理。
7. **コメント解析モジュール (`CommentAnalyzer`)**
   - 受信したコメントを解析（形態素解析、感情判定、スパム判定等）し、結果をシステムに非同期で返す。

## 3. テスト戦略 (GTest / QTest)

システムの品質と拡張性を継続的に担保するため、自動テストを標準で組み込む。

### 3.1 テストフレームワーク
* **ロジック・モジュールテスト**: `GTest` (Google Test) / `GMock` (Google Mock)
* **Qt UI・イベント連携テスト**: `QTest`

### 3.2 テストスコープとアプローチ
* **結合テストのスコープ**: 
  UI側のイベントハンドラ（操作の起点）からスタートし、内部ロジック（解析、DB保存等）を通り、最終的に外部モジュールへ「送信命令」が出されるまでのフローを一気通貫でテストする。
* **外部システムのダミー化 (Mocking / 依存性の注入)**:
  * 実際のTwitchサーバーやOBSと通信するとテストが遅延・不安定になるため、外部連携モジュールは**インターフェース（純粋仮想クラス）**を利用して抽象化する。
  * テスト実行時は、これらを **ダミー（Mockオブジェクト）** に差し替える。
  * 【例】: 実際のHTTPリクエストを飛ばす代わりに、`TwitchActionSender` のダミーへ処理を流し、「NGワードを検知した結果、正しい警告文字列がダミーの送信キューに渡されたか」をGTestのアサートで検証する。

これにより、ネットワーク環境に依存せず、CI環境でも瞬時に実行可能な堅牢な自動テスト環境を実現する。
