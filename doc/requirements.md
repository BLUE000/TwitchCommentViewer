# Twitchコメント解析ツール 要件定義

## 1. 概要
- 目的: Twitchのコメントをリアルタイムに表示・解析し、BOTとして自動応答やチャット連携を行うツール
- 本ツールは、将来的に開発予定のTwitch配信の総合管理ツールにおいて、コメントの表示、解析、BOT機能を提供するモジュールとして流用されることを前提に設計される。
- 対象: 配信者、モデレーター、チャット参加者
- 基本機能:
  - コメント表示（旧コメントビューワ、「コメント」タブ）
  - 視聴者リスト表示（「視聴者」タブ、カテゴリ別分類表示）
  - コメント解析（感情、キーワード、トレンド）
  - BOT機能（自動応答、コマンド処理、モデレーション）

## 2. ステークホルダー
- **配信者**: 主な利用者。コメントのリアルタイム表示や簡単な操作での配信管理を求める。
- **視聴者**: 配信動画を視聴し、コメントを投稿するユーザー。円滑で楽しいチャット体験を期待。
- **モデレーター**: チャットの秩序を保つために荒らしやスパムを管理する役割を持つ。モデレーション機能を活用。
- **開発者 / 運用担当**: システムの設計、開発、保守を担当し、ツールの安定的な運用を実現する。

## 3. システムの範囲とアーキテクチャ
### 技術スタック
- **開発言語 / フレームワーク**: C++ / Qt6
- **対応OS**: Windows先行対応（設計上は将来的なLinux, Mac等のクロスプラットフォーム対応を考慮する）

### 含む機能とモジュール構成
- 各機能はモジュール化され、必要に応じて個別に有効化・無効化が可能であること。
- コメント取得・表示・視聴者リスト機能は「標準機能（コア機能）」として提供する。
- 解析やBOT等の拡張機能は、本体のオプション機能として切り替え可能に実装し、あるいはWebSocket等を利用してOBS等の外部ツールに演出として連携可能な設計とする。
- Twitch APIを使ったコメント取得・視聴者リスト取得機能
- コメント一覧表示機能（「コメント」タブ）
- 視聴者リスト表示機能（「視聴者」タブ、カテゴリ別分類）
- コメント解析機能（日本語対応）
- BOTとしての自動返信・定型応答機能
- コマンド受付・実行機能
- ログ記録と検索機能
- 設定画面 / 設定ファイルによる挙動制御

### 範囲外
- Twitchの配信映像自体の表示
- 他プラットフォーム（YouTube、Discord等）との連携（初期段階ではTwitchに絞る）

## 4. 機能要件
### 4.1 コメント表示＆視聴者リスト
- **コメント表示（「コメント」タブ）**
  - Twitchチャットおよびイベントをリアルタイムに受信して画面表示
  - ユーザー名、コメント内容、タイムスタンプを表示
  - ユーザアカウントに設定されたプロフィールアイコンの画像を表示
  - コメントフィルタ（キーワード、ユーザー、モデレーション）
  - チャンネルポイントの表示および操作（ポイント返却/確定の操作を含む）
  - **Twitch公式チャットへのピン留め機能**
    - コメント行を右クリックし、時間指定（30秒、1分、5分、15分、30分、配信終了まで）でチャット最上部にコメントをピン留め、および解除（アンピン）を行う機能。
  - **Twitchチャットへのアナウンス送信機能**
    - コメントタブの下部にアナウンス用入力欄、色指定プルダウン（通常/primary、青/blue、緑/green、オレンジ/orange、紫/purple）および送信ボタンを配置し、任意のメッセージをチャット欄に目立たせてアナウンス送信する機能。
  - コメント読み上げ機能との連携
    - 棒読みちゃんへのTCPソケット連携、およびVOICEVOX連携（ローカルHTTP API経由）。
    - **共通UIコントロールと個別データ保持**: 設定画面内ではボリューム、速度（スピード）、音程（ピッチ）のUIコントロールをエンジン間で共通化しつつ、設定データは「棒読みちゃん」と「VOICEVOX」で個別に保持・保存する。
    - **読み上げ除外設定**: 「ボット除外」に登録されているユーザーの読み上げを自動でスキップする。また設定画面の「除外ユーザー (カンマ区切り)」に入力された特定のユーザー名も読み上げ対象から除外する。
  - 重要マークなどの操作
- **視聴者リスト（「視聴者」タブ）**
  - Twitch Helix API (`GET /chat/chatters`) を使用して、現在チャットに接続しているユーザーの一覧を取得・一覧表示する。
  - 更新のトリガーと頻度の制限（API負荷低減のため）：
    - **初回表示および自動取得**: 「視聴者」タブの初回表示時、および前回の取得から10分以上経過して再度タブが表示された際（またはタブ表示中に10分経過した際）に自動的に再取得を行う。
    - **手動更新ボタン**: 画面内に「更新」ボタンを配置。前回の取得から3分以上経過している場合のみ有効化され、クリックで手動更新が可能。
    - **非アクティブ時の停止**: 「視聴者」タブが非表示（他のタブが選択されている）のときは、API呼び出しやタイマー監視処理を完全に停止する。
  - 取得したユーザーを以下のカテゴリ（アコーディオン）に分類して一覧表示：
    - すべて: 接続している全ユーザー
    - ストリーマー: 配信者本人
    - モデレーター: モデレーターバッジを持つユーザー
    - VIP: VIPバッジを持つユーザー
    - アーティスト: アーティスト属性を持つユーザー（コメント等から自動検出）
    - チャットボット: 設定でボット除外指定されているユーザー
    - 一般: その他の一般視聴者
    - **重複分類のサポート**: 各カテゴリ分類は排他的ではなく重複を許容する（例: `Nightbot` は「モデレーター」と「チャットボット」の両アコーディオンに分類表示される）。
  - 各リストのストリーマーを除く視聴者行において、マウスホバー時または行選択（クリック）時に、右端に各種アクションボタンを表示する。
    - ボタン構成: [シャウトアウト] [VIP] [モデレータ]  (全角2スペースの余白)  [タイムアウト] [BAN]
    - VIP・モデレータ・BANはトグル動作とし、状態を反映する。モデレーターチャッターに対するモデレーター解除（MODトグルオフ）ができるように保護を解除。
    - 誤操作防止のため、モデレータ追加/削除、タイムアウト（時間設定ダイアログ決定後）、BANの実行時には確認メッセージを表示する。
    - シャウトアウト実行時は2分間のクールタイムが発生し、カウントダウンをウィンドウのタイトルバーに表示する。

### 4.2 コメント解析
- 日本語コメントの形態素解析
- 感情判定（肯定/否定/中立）
- キーワード抽出
- スパム・荒らし判定
- トレンドワードを検出し通知
- ユーザごとのコメント数を集計し、配信者やモデレーターに提供

### 4.3 BOT機能
- 定型応答
- コマンド応答（例: `!help`, `!rules`, `!time`）
- モデレーション機能
  - NGワード自動削除（自動実行または配信者への処理要求の選択が可能）
  - 違反ユーザー警告（自動実行または配信者への処理要求の選択が可能）
- BOTのON/OFF切り替え

### 4.4 設定・管理
- Twitch接続設定（OAuth、チャンネル指定）
- 解析設定（辞書、閾値、除外ワード）
- BOT設定（コマンド登録、応答テンプレート）

### 4.5 OBS演出・物理アバターオーバーレイ機能（追加アセット）
- **物理オーバーレイ表示（OBSブラウザソース用）**
  - **アバターのスポーン**: コメント受信時、そのユーザーのアバターが画面に存在しない場合、画面上端中央から新規アバターを出現させる。既に存在する場合は位置を維持する。
  - **アバターの回転禁止**: アバター画像は常に直立状態（回転角0度固定）を維持する。落下・衝突・移動時も傾きや反転は行わない。
  - **物理演算（落下とバウンド）**: アバターは重力に従って自由落下し、画面下端（地面）で重くバウンドする（軟球・硬球を想定した反発係数 `e = 0.2`〜`0.3` 程度）。回数を重ねるごとに減衰し、最終的に地面に静止する。
  - **吹き出し表示**: コメント受信時にアバターの頭上にコメント内容を吹き出し表示する（アバターの移動に追従）。表示から一定時間（例：7秒）でフェードアウト消滅する。
  - **アバター成長機能**: 同一ユーザーがコメントを重ねるごとに、アバターのサイズをミニマムサイズから段階的に大きく成長させる（上限はマックスサイズ）。サイズ拡大に伴い当たり判定サイズも動的に変更する。
- **コントロール画面（配信主操作用）**
  - **ドラッグ＆ドロップ同期**: 配信主がブラウザで管理画面を開き、アバターをドラッグ＆ドロップで任意の座標に移動できる。
  - **リアルタイム同期**: ドラッグ中はOBS側の物理演算を一時停止し、リアルタイムに位置を同期する。ドロップした瞬間からその座標を起点に物理落下・バウンドを再開する。
- **設定機能**
  - **設定UIの配置場所**: C++ アプリケーションの設定タブ内（`config.json`に設定値を保存）
  - **動的表示制御**: 「オーバーレイファイル」として `overlay_physics.html` が選択されている場合にのみ、本機能の設定項目（グループボックス等）を画面上に表示する。他のファイルが選択されている場合は非表示とする。
  - **調整可能パラメータ**:
    - アバターのミニマムサイズ（スポーン時の初期サイズ、ピクセル単位）
    - アバターのマックスサイズ（成長上限、ピクセル単位）
    - バウンド上限（反発力の設定）：アバターが地面に衝突した際の「跳ね返りやすさ（反発係数）」を0%〜100%の範囲で調整（パーセンテージが高いほど高くバウンドする）。
  - **プレビュー**: 最小サイズと最大サイズが実際にどのくらいの大きさになるかを視覚的に確認できるプレビュー表示をQt UI内に提供する。
- **セキュリティ・製品安全（極めて重要）**
  - **絶対パスの完全拒否と相対パス制御**: C++のHTTPサーバーは、絶対パス（フルパス）の指定を一切拒否する。URL文字制限として、英数字・ハイフン・アンダースコア・スラッシュ・ピリオド以外の文字（特に `%` やスペース、コロン `:`、バックスラッシュ `\`）を含むリクエストは検証を行わずに即座に遮断する。また、canonical path検証により、`assets/overlay/` ディレクトリ配下以外のファイル参照は一切許可しない。
  - **コンテンツセキュリティポリシー（CSP）の強制**: HTTPレスポンスにCSPヘッダーを付与し、ローカル接続およびTwitch公式CDN（画像等）以外の外部ドメインへのデータ送信（情報漏洩）やスクリプト読み込みをブラウザレベルで強制遮断する。
  - **セーフティガバナー（動作保護）**: 画面上の同時物理演算アバター数を制限（例：最大30人）し、1フレームの計算負荷が規定値（例：10ms）を超えた場合は物理演算を自動停止してブラウザ/OBSのフリーズを防ぐ。

## 5. 外部インターフェース要件
- **Twitch連携 (受信)**: チャットやチャンネルポイント等の各種イベントを統合的・リアルタイムに受信するため「EventSub (WebSocket)」を利用する。
- **Twitch連携 (送信)**: BOT機能としての発言送信などは Twitch API (HTTP) を利用する。

## 6. データ・永続化要件
- **管理方式**: 設定やチャット履歴・集計データの管理には、検索性や集計のしやすさを考慮してローカルDB（SQLite等）を利用する。
- **暗号化・難読化**: プライバシー保護の観点から、DBに保存するチャットの発言者や発言内容は難読化ライブラリ（`TransCipher-Dist` 等）を用いて秘匿化する。
- **保持期間とアーカイブ**: チャットデータの保存期間は1ヶ月とし、7日分ごとにまとめて圧縮・アーカイブ化を行う（圧縮するデータは集計等に必要な情報のみを維持する）。

## 7. 画面・UI/UX要件
- **画面構成**: 1つのメインウィンドウ内に「コメント」「視聴者」「解析結果」「設定」等の機能をタブで切り替える方式（タブUI）を採用する。（将来的にモデレーション機能が有効な場合は「モデレーション」タブが追加される。）

## 8. 非機能要件
- **パフォーマンス**: 1000コメント/分程度の処理を遅延なく表示
- **信頼性・可用性・エラーハンドリング**:
  - Twitch API切断時の再接続機能
  - セッション切断や再接続成功など、自動復帰が見込めるステータス変化は「画面下部のステータスバー」に表示する。
  - トークン無効、ユーザ設定ミス、アカウントBANなど、ユーザの操作なしには復帰できない問題については「ダイアログ」を表示して警告する。
- **拡張性・モジュール性**: 
  - 将来の他プラットフォーム追加や解析機能追加が容易な設計とする。
  - 将来開発する「総合管理ツール」で再利用可能な設計とする。
- **セキュリティ (OAuthトークン管理)**:
  - OAuthトークンなどの機密情報は平文で保存しない。
  - WindowsのDPAPI機能や、`QtKeychain` 等のOSの資格情報管理機構を利用し、OSのログインユーザのみが復号できる安全な形式で保存する。
- **運用性**: 設定のエクスポート/インポート、ログの参照性

## 9. 制約
- Twitch APIのレート制限に従う
- 日本語チャット向け形態素解析ライブラリを利用

## 10. 受け入れ基準
- Twitchのチャットやイベントが正常に取得・表示できる
- 指定コマンドに対してBOTが正しく応答する
- チャンネルポイントが正しく受信・操作できる
- 解析結果が画面に表示される
- 設定変更が反映される
- 認証情報が暗号化されて安全に保存・読み込みできている

- システム設計: アーキテクチャ、モジュール構成、データフロー
- 詳細設計: 画面レイアウト、API設計、データ構造、DBスキーマ設計
