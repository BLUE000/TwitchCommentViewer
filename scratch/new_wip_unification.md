## 開発時の重要注意事項（エンコーディング・編集ルール）

AIアシスタントなどの自動ツールによるファイル編集の際、文字化けを防ぐために以下のルールを遵守すること：
- **PowerShellの Get-Content/Set-Content を使用してファイル編集を行わないこと。**（Windowsの既定エンコーディングによる上書き破損を防ぐため）
- **ファイル編集は直接行わず、文字コードを UTF-8 に固定した適切なエディタやスクリプトを使用すること。**

## 今後の機能拡張アイデア

### 1. 翻訳機能 (Translation)
- **概要**: 取得したコメントを別の言語に翻訳する機能。
- **想定仕様**: 
  - 設定タブにてON/OFF可能。
  - DeepL API または Google Translate API 等を利用.
  - 英語コメントを日本語に翻訳、または日本語コメントを英語にしてOBSに表示.

### 2. モデレーション機能の簡易化 (Moderation GUI)
- **概要**: Twitchのダッシュボードを開かずに、ツール上から荒らし対策やチャット管理をワンクリックで行う機能。
- **想定仕様**:
  - 新たな専用タブ（「モデレーション」タブなど）を追加。
  - **ユーザーリスト表示**: 直近で発言したユーザーをリストアップ。
  - **ワンクリックアクション**: ユーザー名を右クリック等で、「10分タイムアウト」「BAN」「解除}などのメニューを即座に実行（Twitch APIを利用）。
  - **スパム自動連携**: 既存のコメント解析（スパム判定）で引っかかったユーザーをリスト内でハイライトし、視覚的に分かりやすくする。

## メモ
- 新しい機能を追加する際は、必ず ConfigManager にフラグを持たせ、設定タブからON/OFFを切り替えられる設計（モジュール化）を維持する。
- 機能をOFFにした場合は、バックグラウンドでの処理負荷も完全に停止するように考慮すること。
